Best Practices Using ID and Class Selectors

February 25, 2016

Using IDs and Classes

IDs and classes are selectors used in CSS to target specific HTML elements. By targeting the elements, we can then change them and make them look stylish and fancy. Let's take a look at the buttons on the left as example. The default color was black and the size was much smaller, so I used classes to target all the buttons and made them larger but I also used classes to target each of the buttons individually and changed the color.

Here are the steps to using IDs and classes:

  • IDs
    1. We add id="id-name" to the HTML page, with id-name as an example name
    2. Let's say we want to add an ID to p element, it'll look like this: <p id="id-name"> (include the quotes)
    3. Then we target the specific p element by using #id-name on the CSS page
    4. It will look like this: #id-name { color: blue; }
  • Classes
    1. We add class="class-name" to the HTML page, with class-name as an example name
    2. Let's say we want to add a class to p element, it'll look like this: <p class="class-name"> (include the quotes)
    3. Then we target the specific p element by using .class-name on the CSS page
    4. It will look like this: .class-name { color: blue; }

Notice we used # (hash tag) to target IDs and . (period) to target classes. So what's the difference between using IDs and classes? There are a few rules that differentiates them:

  • ID's are unique
    • Each element can have only one ID
    • Each page can have only one element with that ID
  • Classes are NOT unique
    • Each element can have multiple classes
    • Each class can be used on multiple elements

Barcodes vs Serial Numbers

A good analogy I found is comparing barcodes and serial numbers. Let's take your computer for example, it has a barcode and when scanned it will show the computer's price. This same barcode is used for all computers that fit the same type. On the other hand, your computer has a unique serial number which distinguishes it to another computer of the same type. Now if we want to change the price, we could update each individual serial number or just update the one barcode. This is just like ID and classes.

How it all Comes Together

  • IDs and classes are selectors
  • IDs are unique while classes are not
  • Elements can have both IDs and classes
  • Simply adding an ID or class does nothing to the element (needs CSS to target them and style)
  • Information that is reuseable should be kept in class while information that is unique can be kept in an ID

Since classes can be used again and again for more than one element, most people suggest to use classes unless you need to target the element using JavaScript selectors (we'll get to that in future posts) or you want the browser to locate an element and automatically scroll to the element like https://billdevcode.github.io/#about
