What is Command Line? Why is it useful and how can a beginner learn it?

Most people are familiar with wallpapers, folders, icons, using the mouse and double-clicking. This is called graphical user interface (GUI) which is shown on the right side of the image. You use these tools to navigate and control your computer. It's visually intuitive, simple to learn and use, you can open multiple programs and easily jump between the programs with a mouse.

Another way to navigate and control your computer is command line interface (CLI) which is shown on the left side of the image. With command line you will only see text, use the keyboard and almost never touch the mouse. Since you have to memorize and use specific commands to do all the things you would normally do in GUI, it's generally harder to learn CLI. And not to mention some people are probably turned-off by all the text.

Although there some things you can do faster using GUI than CLI, for example switching between multiple programs, there are many things you can do with CLI which are almost impossible to do with GUI for example, list all the files you modified this month. Here is an interesting article that talks about the pros and cons of using GUI and CLI which you can find here.

I think GUI and CLI are both useful for different purposes so it's a good idea to learn both. If you're looking for a good resource to learn the command line check out the command line crash course.

What is Git? What is Github? Why are they useful?

Git is a free and open source version control system mostly used for software development. Version control is when you organize many versions and configurations of a software. I'm convinced version control is a wonderful tool to have and Git is one of the popular version control tools. Here are some benefits of using version control (Git):

  • At any given time, you can see how much work is being done, which part of the source code is being worked on, when changes were made and by whom
  • If you made changes and realized it was a mistake, you can revert back
  • Helps you automatically keep track and organize multiple versions of your codes so you can focus on the more important tasks
  • Not convinced and want more reasons to use version control? Check out Stack Overflow

Github is a web-based Git repository hosting service. Think of it as Git but on the web instead of on your computer with a few other features. An alternative to Github is BitBucket but what makes Github so powerful are 3 key features according to TechCrunch:

  1. Fork - copying a repository from one account to another, essentially taking someone else's project (with their permission)
  2. Pull request - requesting to have your changes added to the master project
  3. Merge - accepting a pull request and merging the new changes to the master project

These are the things I am learning this week: command line, Git, and Github, all of which developers probably use every day. I think it really helps to learn these tools and best practices from the start so we can get into the mindset of being a developer.
