Arrays and Hashes

March 3, 2016

Arrays

Arrays are ordered, number-indexed collections of any object or in short, a list. Arrays are used to store things (objects) in an ordered list such as a list of songs, a list of students in a classroom, or a list of keystrokes a user clicks. Here are some examples of arrays:

  1. [ ]
    • This is an empty array
  2. [5]
  3. ["hello", "world"]
    • This is an array with two objects separated by the comma. hello is on index 0 and world is on index 1
  4. ["hello", 1, 2, 3, true, false, [address, "san francisco"]]

Arrays are enclosed in the [ ] brackets, separated by the commas and the first index starts with 0 (zero) then 1, 2, 3, etc. Arrays are very useful in keeping track of an ordered list of things. Learn more about Arrays

Hashes

Hashes are dictionary-like collection of unique keys and their values. Hashes are also used to store things (objects) similar to arrays but hashes doesn't necessarily have to store things using number-indexes instead hashes can use any object (instead of numbers). Here are some examples of hashes:

  1. hash_example[:first_name]
    • We are looking up the value of :first_name in a hash named hash_example
  2. options = { font_size: 10, font_family: "Arial" }
  3. grades = { "Jane Doe" => 10, "Jim Doe" => 6 }
    • The hash is named grades
    • The keys are Jane Doe and Jim Doe
    • The values are 10 and 6
    • The => sign is used to declare a hash object.

Hashes are enclosed in the { } brackets, also separated by commas. We lookup a value by using it's key. Hashes can be used to store just about anything like a birthday list (names and dates), your bank balance (names and balance), even the indentities of your favorite superheroes (superhero identity and real identity), etc. Learn more about Hashes

Conclusion

Arrays and hashes are both useful in the real-world. If you want a simple ordered list that's organized in numbers starting from index 0 then an array is what you're looking for. If you want a dictionary-like list that you can lookup things and not have to use numbers only then a hash is what you're looking for.
